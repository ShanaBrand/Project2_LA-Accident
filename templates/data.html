Logic of Analysis






our toolkit:

》Python, Flask, Restful API, HTML/CSS, Plotly

》Javascript, NPM, MySQL, D3, Leaflet.js.

our data source:

》"Traffic Collision Data from 2010 to Present" from the City of Los Angeles Data Catalog.

》Data Owner: LAPD

》Refresh Rate: Weekly

our lessons:

》Public/raw datasets must be thoughoughly cleaned before using.

》Leaflet.js is sensitive with the source format for Latitude and Longitude in the database.







I.

Clean the Data.



We renamed columns, removed appropriate null values, removed lat/longs of 0.

II.

Analyze with D3 + Plotly + Leaflet.js

We loaded in the API from the data source for a GeoJSON. Then we constructed a Heatmap with Leaflet.js, to illustrate the most populated reported traffic collisions. In addition to the Heatmap, we included an interactive Leaflet.js mapping of the collision sites with markers that that pop-up, with relevant details on that specific incident, when clicked. For the Plotly + D3 portion of this project, we explored 4 main subsets in the data: victim age, victim descent, victim gender and frequency of incidents/year.

III.

Structure Flask Routes for Web Hosting

We first set up a shared repo to ensure everyone has access to all updates and changes. Then, we defined the table structure before importing the data. With the Flask routes, we have set "/" as our INDEX or landing page. You will be able to navagate through the site via the NavBar as well as the many other interactive features provided.

IV.

Load and Run

This last step is always the most exciting, as you can see.

Columns in this Dataset

DR NUMBER:

Data Type (Text), API Field Name (dr_no)

Division of Records Number: Official file number made up of a 2 digit year, area ID, and 5 digits.

DATE REPORTED:

Data Type(Floating Timestamp), API Field Name(date_rptd)

MM/DD/YYYY

DATE OCCURRED:

Data Type(Floating Timestamp), API Field Name(date_occ)

MM/DD/YYYY

TIME OCCURRED:

Data Type (Text), API Field Name (time_occ)

In 24 hour military time.

AREA ID:

Data Type (Text), API Field Name (area)

The LAPD has 21 Community Police Stations referred to as Geographic Areas within the department. These Geographic Areas are sequentially numbered from 1-21.

AREA NAME:

Data Type (Text), API Field Name (area_name)

The 21 Geographic Areas or Patrol Divisions are also given a name designation that references a landmark or the surrounding community that it is responsible for. For example 77th Street Division is located at the intersection of South Broadway and 77th Street, serving neighborhoods in South Los Angeles.

REPORTING DISTRICT:

Data Type (Text), API Field Name (rpt_dist_no)

A code used in producing reports to group data into geographic sub-areas within an area. Find LAPD Reporting Districts on the LA City GeoHub.

CRIME CODE:

Data Type (Text), API Field Name (crm_cd)

Indicates the crime committed. For this dataset - all Crime Code 997.

CRIME CODE DESCRIPTION:

Data Type (Text), API Field Name (crm_cd_desc)

Defines the Crime Code provided. For this dataset - all Traffic Collision.

MO CODES:

Data Type (Text), API Field Name (mocodes)

Modus Operandi: Activities associated with the suspect in commission of the crime. See attached PDF for list of MO Codes in numerical order.

VICTIM AGE:

Data Type (Text), API Field Name (vict_age)

Two character numeric.

VICTIM SEX:

Data Type (Text), API Field Name (vict_sex)

F - Female M - Male X - Unknown

VICTIM DESCENT:

Data Type (Text), API Field Name (vict_descent)

Descent Code: A - Other Asian B - Black C - Chinese D - Cambodian F - Filipino G - Guamanian H - Hispanic/Latin/Mexican I - American Indian/Alaskan Native J - Japanese K - Korean L - Laotian O - Other P - Pacific Islander S - Samoan U - Hawaiian V - Vietnamese W - White X - Unknown Z - Asian Indian

PREMISE CODE:

Data Type (Text), API Field Name (premis_cd)

The type of structure or location where the incident took place.

PREMISE DESCRIPTION:

Data Type (Text), API Field Name (premis_desc)

A code used in producing reports to group data into geographic sub-areas within an area. Find LAPD Reporting Districts on the LA City GeoHub.

ADDRESS:

Data Type (Text), API Field Name (location)

Street address of crime incident rounded to the nearest hundred block to maintain anonymity.

CROSS STREET:

Data Type (Text), API Field Name (cross_street)

Cross Street of rounded Address.

LOCATION:

Data Type (Location), API Field Name (location_1)

The location where the crime incident occurred. Actual address is omitted for confidentiality. XY coordinates reflect the nearest 100 block.
